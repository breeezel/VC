# VoiceClonerPy - Измени Свой Голос!

## Что это за программа?

VoiceClonerPy — это программа, которая может изменять ваш голос. Представьте, что вы можете говорить голосом другого человека или создавать совершенно новый, уникальный голос! Эта программа использует продвинутые технологии искусственного интеллекта, чтобы научиться имитировать различные голоса и применять эти знания для преобразования вашего собственного голоса.

Вы можете использовать ее для:
*   Превращения вашего голоса в голос другого диктора (если у вас есть образцы его голоса).
*   Создания уникального голоса путем "обучения" программы на вашем собственном голосе (это называется "тонкая настройка" или "fine-tuning").
*   Изменения голоса в уже записанных аудиофайлах.
*   (В будущем) Изменения голоса в реальном времени, например, во время разговора.

## Основные возможности

*   **Обучение на многих голосах**: Программа может учиться на больших наборах данных с разными голосами, чтобы понимать общие принципы изменения голоса.
*   **Настройка на ваш голос**: Вы можете "научить" программу вашему уникальному голосу, используя всего несколько минут ваших аудиозаписей.
*   **Изменение голоса в файлах**: Загрузите аудиофайл, и программа изменит голос в нем.
*   **Изменение голоса во время разговора (в разработке)**: В будущем программа сможет изменять ваш голос прямо во время разговора по микрофону.
*   **Простая настройка через файл**: Основные параметры можно менять в специальном текстовом файле (конфигурации).

## Установка программы

Чтобы программа заработала на вашем компьютере, нужно выполнить несколько шагов. Это может показаться сложным, но если следовать инструкции, все получится!

1.  **Скачайте программу (если это архив) или скопируйте файлы проекта:**
    Если вы скачали архив, распакуйте его в удобную папку. Если это проект на GitHub, его нужно "клонировать". Для этого используется команда:
    ```bash
    # git clone https://github.com/ваш_username/VoiceClonerPy.git
    # cd VoiceClonerPy
    ```
    *Примечание: Эти команды для тех, кто знаком с Git. Если вы просто скачали папку с файлами, перейдите к следующему шагу.*

2.  **Установите Python (если еще не установлен):**
    Программе нужен Python для работы. Если у вас его нет, скачайте и установите с [официального сайта Python](https://www.python.org/downloads/). Рекомендуется версия Python 3.8 или новее.

3.  **(Рекомендуется) Создание "виртуального пространства":**
    Это как отдельная коробочка для нашей программы, чтобы она не мешала другим программам на компьютере. Откройте командную строку (терминал) в папке с программой и введите:
    ```bash
    # python -m venv venv
    # source venv/bin/activate  # для Linux/macOS
    # venv\Scripts\activate    # для Windows
    ```
    *Примечание: Эти команды выполняются пользователем в его терминале. Если сложно, этот шаг можно пропустить, но он помогает избежать проблем в будущем.*

4.  **Установите необходимые "библиотеки":**
    Это как дополнительные инструменты, которые нужны программе. В папке с программой есть файл `requirements.txt`, где они перечислены. Установите их командой:
    ```bash
    pip install -r requirements.txt
    ```
    Возможно, потребуется отдельно установить библиотеку PyTorch. Инструкции есть на [сайте PyTorch](https://pytorch.org/). Выберите вашу операционную систему и способ установки (обычно с Pip).

5.  **(Опционально) Установка `espeak-ng`:**
    Эта программа может понадобиться для некоторых продвинутых функций, связанных с анализом речи. Если она потребуется, ее можно установить так:
    ```bash
    # sudo apt-get install espeak-ng  # для Debian/Ubuntu Linux
    ```
    Для Windows и macOS установка будет другой.

## Подготовка ваших аудиозаписей

Качество работы программы сильно зависит от качества и количества ваших аудиозаписей.

1.  **Для "настройки на ваш голос" (Fine-tuning):**
    *   **Что нужно?** Запись вашего голоса.
    *   **Качество:** Говорите четко, без посторонних шумов (эхо, музыка, разговоры на фоне). Используйте хороший микрофон.
    *   **Длительность:** От 15 до 60 минут чистого голоса. Чем больше, тем лучше может быть результат.
    *   **Формат:** Сохраните аудио в формате WAV.
    *   **Где хранить:** Создайте отдельную папку на вашем компьютере и положите туда все WAV файлы с вашим голосом. Путь к этой папке нужно будет указать в файле настроек.

2.  **Для обучения "с нуля" на многих голосах (если вы разработчик):**
    *   Нужен большой набор данных с разными дикторами (например, VCTK-Corpus).
    *   Это сложный процесс и обычно не требуется обычному пользователю.

## Настройка программы (файл `config/config_template.yaml`)

Программа настраивается через текстовый файл `config/config_template.yaml`. Вы можете скопировать его, назвать, например, `my_settings.yaml`, и изменять копию.

**Важно:** Изменяйте только те параметры, которые вы понимаете. Некоторые настройки предназначены для разработчиков, и их изменение без понимания может привести к ошибкам.

Вот некоторые важные параметры, сгруппированные по задачам:

**1. Общие настройки проекта (`project`):**
*   `experiment_name`: Название вашего "эксперимента" или проекта. Помогает организовать сохраняемые файлы. (например, "moi_golos_test1")

**2. Настройки аудиоданных (`data`):**
*   `fine_tune_user_data_dir`: **Очень важно для настройки на ваш голос!** Укажите здесь путь к папке с вашими WAV файлами (см. "Подготовка ваших аудиозаписей").
*   `sample_rate`: Частота дискретизации. Обычно для голоса достаточно 22050 или 24000. Убедитесь, что ваши аудиофайлы имеют такую же частоту.

**3. Настройки для конкретных задач (в секции `training` и других):**
*   `training.mode`: Режим работы программы. Вы будете менять его в зависимости от того, что хотите сделать (см. "Как пользоваться программой").
*   `model.num_speakers`: Если вы обучаете программу на нескольких дикторах (продвинутый режим), здесь указывается их количество. Для настройки на один ваш голос, это обычно 1 (или 2, если используется предобученная базовая модель).
*   `training.checkpoint_dir`: Папка, куда будут сохраняться "мозги" программы (обученные модели).
*   `training.log_file_path`: Файл, куда будут записываться технические сообщения о работе программы.

**4. Настройки для изменения голоса в файле (`inference_file`):**
*   `input_wav_path`: Путь к аудиофайлу, голос в котором вы хотите изменить.
*   `output_wav_path`: Куда сохранить результат.
*   `conversion_direction`: Направление изменения (например, 'specific', если вы используете модель, настроенную на ваш голос, или 'male_to_female').
*   `specific_generator_checkpoint_path`: Путь к файлу модели генератора (обычно заканчивается на `..._generator.pth`), который будет использоваться для изменения голоса.
*   `specific_target_speaker_id`: ID (номер) целевого голоса, в который нужно преобразовать. Обычно 0, если модель настроена на один конкретный голос.

**5. Настройки для изменения голоса во время разговора (`inference_realtime`):**
*   `input_device_index`, `output_device_index`: Номера вашего микрофона и наушников/динамиков. Если оставить -1, программа предложит выбрать их при запуске.
*   Остальные параметры похожи на `inference_file` (выбор модели, целевого голоса).

**Предупреждение:** Остальные параметры в файле конфигурации (например, `generator`, `discriminator`, `lambda_...`) являются техническими. Их изменение без глубокого понимания работы нейронных сетей не рекомендуется.

## Как пользоваться программой

Все действия выполняются через одну команду в терминале (командной строке), но с разными настройками.

**Основная команда:**
```bash
python scripts/run.py --config путь/к/вашему/файлу_настроек.yaml --mode режим
```

**Примеры режимов:**

### 1. Настройка программы на ваш уникальный голос (`fine_tune`)

Это самый частый сценарий, если вы хотите, чтобы программа "говорила" вашим голосом или голосом, на который вы ее обучите.

1.  **Подготовьте аудио**: Запишите свой голос (15-60 минут, WAV, хорошее качество) и положите файлы в отдельную папку.
2.  **Настройте файл конфигурации** (например, `config/my_voice_finetune.yaml`):
    *   `data.fine_tune_user_data_dir`: Укажите путь к папке с вашими записями.
    *   `project.experiment_name`: Придумайте имя для этого эксперимента, например, "moi_golos_na_Kate".
    *   `model.num_speakers`: Если вы используете предобученную базовую модель, это число должно соответствовать ей. Если обучаете только на своем голосе с нуля (не рекомендуется для начала), можно поставить 1 (но это потребует изменений в других частях конфига). Для простоты, предположим, что вы дообучаете модель, которая уже знает о разных дикторах, и ваш голос будет одним из них.
    *   В секции `fine_tuning`:
        *   `base_model_checkpoint_path`: Путь к файлу **генератора** базовой модели (если вы дообучаете). Если у вас нет базовой модели, этот этап сложнее и требует сначала обучения базовой модели.
        *   `num_epochs`: Количество "проходов" обучения. Для начала можно поставить 100-500.
        *   `learning_rate_g`: Скорость обучения. Для fine-tuning обычно маленькая, например, 0.00005.
    *   `training.mode`: Установите в `fine_tune`.
3.  **Запустите настройку:**
    ```bash
    python scripts/run.py --config config/my_voice_finetune.yaml --mode fine_tune
    ```
    Процесс может занять время. Следите за сообщениями в консоли. Модель будет сохраняться в папку, указанную в `training.checkpoint_dir`.

### 2. Изменить голос в аудиофайле (`inference_file`)

1.  **У вас должна быть обученная модель** (либо базовая, либо настроенная на ваш голос).
2.  **Настройте файл конфигурации** (например, `config/convert_audio.yaml`):
    *   `training.mode`: Установите в `inference_file`.
    *   В секции `inference_file`:
        *   `input_wav_path`: Путь к аудио, которое хотите изменить.
        *   `output_wav_path`: Куда сохранить результат.
        *   `specific_generator_checkpoint_path`: Путь к файлу вашей обученной модели генератора (например, `checkpoints/moi_golos_na_Kate/model_best_mcd_generator.pth`).
        *   `specific_target_speaker_id`: ID диктора, в чей голос конвертировать. Если модель обучалась на одном вашем голосе (или вы дообучали на свой голос с ID 0 в рамках многопользовательской модели), то здесь обычно 0.
    *   Убедитесь, что `model.vocoder.checkpoint_path` указывает на рабочий файл вокодера (см. "Что ожидать").
3.  **Запустите изменение голоса:**
    ```bash
    python scripts/run.py --config config/convert_audio.yaml --mode inference_file
    ```

### 3. Изменять голос во время разговора (`inference_realtime`)

1.  **У вас должна быть обученная модель.**
2.  **Настройте файл конфигурации** (например, `config/realtime_conversion.yaml`):
    *   `training.mode`: Установите в `inference_realtime`.
    *   В секции `inference_realtime`:
        *   `input_device_index`, `output_device_index`: ID вашего микрофона и наушников/динамиков. Если оставить -1, программа спросит при запуске.
        *   `specific_generator_checkpoint_path` и `specific_target_speaker_id`: Как в режиме `inference_file`.
    *   Убедитесь, что `model.vocoder.checkpoint_path` указывает на рабочий файл вокодера.
3.  **Запустите программу:**
    ```bash
    python scripts/run.py --config config/realtime_conversion.yaml --mode inference_realtime
    ```
    Если ID устройств не указаны, программа покажет список и попросит их ввести. Для остановки нажмите Ctrl+C в консоли.

### 4. Обучение программы на множестве голосов (`train`)
Этот режим предназначен для разработчиков или продвинутых пользователей, которые хотят обучить базовую модель с нуля на большом датасете (например, VCTK). Он требует значительных вычислительных ресурсов и времени. Настройки аналогичны `fine_tune`, но используются параметры из секции `training` (например, `data.base_model_corpus_dir`, более высокие `num_epochs` и `learning_rate_g`).

## Качество Результата

Качество преобразованного голоса зависит от многих факторов:
*   **Качество и количество аудиоданных**: Чистые записи без шума и достаточной длительности очень важны.
*   **Время обучения/настройки**: Чем дольше программа учится (в разумных пределах), тем лучше может быть результат.
*   **Параметры модели и обучения**: Технические настройки, которые обычно подбираются экспериментально.
*   **Вокодер**: Это отдельная модель, которая превращает внутреннее представление голоса (мел-спектрограмму) обратно в слышимый звук. Качество вокодера сильно влияет на итоговый результат.

## Что ожидать и что пока не работает идеально (Ограничения и TODO)

*   **Качество звука и вокодер**: Для наилучшего качества звука программе нужен специальный файл-модель вокодера (например, HiFi-GAN). Если он не настроен или отсутствует, программа будет использовать очень упрощенный способ генерации звука, который может звучать роботизированно или как синусоида. Настройка и использование качественного вокодера — важный шаг.
*   **Измерение качества**: Функции для точного измерения "похожести" голосов (MCD, F0-RMSE) пока являются "заглушками" и не выполняют реальных вычислений.
*   **Изменение голоса в реальном времени**: Эта функция требует очень быстрой работы всех компонентов и сложной обработки звука "на лету". Текущая реализация — это только основа, и она может работать с большой задержкой или не так гладко, как хотелось бы.
*   **Подготовка данных**: Загрузка и подготовка данных для обучения (особенно для больших наборов) пока упрощена.
*   **Сложные сценарии с ID дикторов**: Если вы хотите добавить много новых голосов или управлять ими сложным образом, текущая система эмбеддингов может потребовать доработок.

Этот проект — это инструмент для экспериментов. Не бойтесь пробовать разные настройки (особенно те, что описаны как безопасные для пользователя), но будьте готовы, что для достижения идеального результата может потребоваться время и терпение!
