# VoiceClonerPy - Клонирование Голоса на Python

## Введение

VoiceClonerPy - это система для клонирования и преобразования голоса, основанная на модели StarGAN-VC (Generative Adversarial Network for Voice Conversion). Проект позволяет обучать модели для преобразования голоса одного диктора в голос другого, а также выполнять тонкую настройку (fine-tuning) на небольшом количестве аудиоданных для имитации конкретного голоса.

Цель проекта - предоставить гибкий и расширяемый инструмент для экспериментов с преобразованием голоса, поддерживающий различные этапы: от подготовки данных и обучения модели до инференса (преобразования голоса) в реальном времени или из файлов.

## Возможности

*   **Обучение базовой модели StarGAN-VC**: Позволяет обучать модель на большом корпусе данных с множеством дикторов.
*   **Тонкая настройка (Fine-tuning)**: Адаптация предварительно обученной модели к голосу конкретного пользователя с использованием небольшого объема аудио (несколько минут).
*   **Преобразование голоса из файла**: Конвертация аудиофайла с голосом одного диктора в голос целевого диктора.
*   **Преобразование голоса в реальном времени (Real-time)**: Захват аудио с микрофона, преобразование и воспроизведение в реальном времени (структура заложена, требует полной реализации потоковой обработки).
*   **Конфигурируемость**: Гибкая настройка параметров проекта, данных, моделей и обучения через YAML файлы.
*   **Модульная структура**: Код организован в модули для упрощения понимания, поддержки и расширения.
*   **Оценка качества**: Предусмотрена возможность интеграции метрик для оценки качества преобразования (например, MCD, F0-RMSE), на данный момент с плейсхолдерами.

## Установка

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/ваш_username/VoiceClonerPy.git
    cd VoiceClonerPy
    ```

2.  **Создайте и активируйте виртуальное окружение (рекомендуется):**
    ```bash
    # python -m venv venv
    # source venv/bin/activate  # для Linux/macOS
    # venv\Scripts\activate    # для Windows
    ```
    *Примечание: Команды создания и активации виртуального окружения выполняются пользователем в его терминале.*

3.  **Установите зависимости:**
    Основные зависимости перечислены в файле `requirements.txt`. Установите их с помощью pip:
    ```bash
    pip install -r requirements.txt
    ```
    Вам также может потребоваться установить PyTorch с поддержкой CUDA отдельно, в зависимости от вашей системы. Инструкции см. на [официальном сайте PyTorch](https://pytorch.org/).

4.  **Установка Espeak NG (для фонемного анализатора, если будет использоваться):**
    Для некоторых продвинутых функций (например, фонемный ввод для синтеза) может потребоваться `espeak-ng`.
    ```bash
    sudo apt-get install espeak-ng  # для Debian/Ubuntu
    ```

## Подготовка Данных

1.  **Базовая модель**: Для обучения основной модели преобразования голоса требуется большой корпус данных с множеством дикторов. Примером такого корпуса является VCTK.
    *   Скачайте и распакуйте корпус.
    *   Укажите путь к аудиофайлам в `config.<ваша_конфигурация>.yaml` в секции `data.base_model_corpus_dir`.
    *   Данные должны быть в формате WAV. Рекомендуется предварительная обработка (например, удаление тишины, нормализация громкости).

2.  **Тонкая настройка (Fine-tuning)**:
    *   Запишите несколько минут аудио (5-10 минут) от пользователя, чей голос вы хотите клонировать.
    *   Сохраните аудио в формате WAV в отдельную папку.
    *   Укажите путь к этой папке в `config.<ваша_конфигурация>.yaml` в секции `data.fine_tune_user_data_dir`.

## Конфигурация (`config/config_template.yaml`)

Все параметры проекта управляются через YAML файлы в папке `config/`. Основной шаблон `config_template.yaml` содержит следующие секции:

*   **`project`**: Общие настройки проекта.
    *   `name`: Название проекта.
    *   `version`: Версия проекта.
    *   `description`: Краткое описание.
    *   `authors`: Авторы.
    *   `experiment_name`: Уникальное имя для конкретного эксперимента (используется для именования папок с чекпоинтами и логами).

*   **`data`**: Параметры, связанные с данными и их обработкой.
    *   `base_model_corpus_dir`: Путь к директории с аудиоданными для обучения базовой модели (например, корпус VCTK).
    *   `fine_tune_user_data_dir`: Путь к директории с аудиоданными пользователя для тонкой настройки.
    *   `val_data_dir`, `val_metadata_file`: Пути к данным и метаданным для валидации.
    *   `sample_rate`: Целевая частота дискретизации для всех аудио.
    *   `n_fft`, `hop_length`, `win_length`, `n_mels`, `fmin`, `fmax`: Параметры для вычисления мел-спектрограмм (STFT и Mel фильтры).
    *   `power_to_db`: Преобразовывать ли мощность спектрограммы в децибелы.
    *   `trim_silence`, `trim_top_db`: Параметры для удаления тишины в начале и конце аудиофайлов.
    *   `batch_size`, `num_workers`, `pin_memory`: Стандартные параметры для `DataLoader` PyTorch.
    *   `segment_length_sec`: Длина аудио сегментов в секундах, на которые нарезаются треки для обучения.

*   **`model`**: Настройки архитектуры модели.
    *   `type`: Тип основной модели (например, "StarGANVC").
    *   `num_speakers`: Общее количество уникальных дикторов в датасете для обучения базовой модели.
    *   `speaker_embedding_dim`: Размерность вектора эмбеддинга диктора.
    *   `generator`: Параметры архитектуры генератора (количество каналов в слоях, количество блоков и т.д.).
    *   `discriminator`: Параметры архитектуры дискриминатора.
    *   `vocoder`: Настройки вокодера (например, HiFi-GAN).
        *   `type`: Тип вокодера.
        *   `checkpoint_path`: Путь к предобученному чекпоинту вокодера.
        *   `config_path`: Путь к конфигурационному файлу вокодера (если требуется).

*   **`training`**: Параметры процесса обучения.
    *   `mode`: Режим работы скрипта (`train`, `fine_tune`, `inference_file`, `inference_realtime`).
    *   `num_epochs`: Количество эпох обучения.
    *   `learning_rate_g`, `learning_rate_d`: Скорости обучения для генератора и дискриминатора.
    *   `optimizer_betas`: Параметры беты для оптимизатора Adam.
    *   `checkpoint_dir`: Директория для сохранения чекпоинтов модели.
    *   `log_file_path`: Путь к файлу логов обучения.
    *   `log_interval`: Как часто (в шагах/батчах) выводить информацию о процессе обучения.
    *   `save_epoch_interval`: Как часто (в эпохах) сохранять чекпоинт модели.
    *   `resume_from_epoch`: Номер эпохи (1-индексированный) для возобновления обучения с чекпоинта. 0 - не возобновлять.
    *   `lambda_g_adv`, `lambda_g_cls`, `lambda_identity`, `lambda_reconstruction`: Веса для различных компонент функции потерь генератора.
    *   `lambda_d_adv`, `lambda_d_cls`: Веса для функций потерь дискриминатора.
    *   `device`: Устройство для обучения (`cuda` или `cpu`).
    *   `batch_size_val`: Размер батча для валидации.

*   **`fine_tuning`**: Параметры, специфичные для этапа тонкой настройки.
    *   `base_model_checkpoint_path`: Путь к чекпоинту генератора предобученной базовой модели.
    *   `num_epochs`, `learning_rate_g`, `learning_rate_d`: Количество эпох и скорости обучения для тонкой настройки (обычно меньше, чем для базового обучения).
    *   `train_generator`, `train_discriminator`: Флаги, указывающие, какие части модели обучать.
    *   `lambda_identity_ft`, `lambda_reconstruction_ft`: Веса потерь, специфичные для fine-tuning (могут отличаться от основного обучения).

*   **`inference_file`**: Настройки для преобразования голоса из файла.
    *   `input_wav_path`: Путь к исходному аудиофайлу.
    *   `output_wav_path`: Путь для сохранения сконвертированного аудиофайла.
    *   `conversion_direction`: Направление конверсии ('male_to_female', 'female_to_male', 'specific').
    *   `fine_tuned_model_path_female_generator`, `fine_tuned_model_path_male_generator`: Пути к генераторам, дообученным на женский и мужской голос соответственно.
    *   `specific_generator_checkpoint_path`, `specific_target_speaker_id`: Путь к конкретной модели генератора и ID целевого диктора, если `conversion_direction` установлено в 'specific'.
    *   `target_female_speaker_id`, `target_male_speaker_id`: ID целевых дикторов для соответствующих направлений конверсии.

*   **`inference_realtime`**: Настройки для преобразования голоса в реальном времени.
    *   `input_device_index`, `output_device_index`: ID устройств ввода/вывода (-1 для интерактивного выбора).
    *   `noise_suppression_enabled`: Включить ли подавление шума.
    *   `buffer_size_samples`: Размер буфера для аудиопотока в сэмплах.
    *   Параметры выбора модели и целевого диктора, аналогичные `inference_file`.

*   **`evaluation`**: Параметры для объективной оценки.
    *   `primary_metric`: Основная метрика для сохранения лучшей модели (например, 'mcd').
    *   `mfcc_params`: Параметры для извлечения MFCC (для MCD).
    *   `f0_params`: Параметры для извлечения F0 (для F0-RMSE).

## Использование

Основной точкой входа для всех операций является скрипт `scripts/run.py`.

**Общий формат запуска:**
```bash
python scripts/run.py --config path/to/your_config.yaml --mode <mode_name>
```

### Обучение Базовой Модели

1.  **Настройте конфигурационный файл** (например, `config/train_base_config.yaml`):
    *   `data.base_model_corpus_dir`: Укажите путь к вашему датасету (например, VCTK).
    *   `data.val_data_dir`, `data.val_metadata_file`: Укажите пути к валидационным данным (если есть).
    *   `model.num_speakers`: Установите корректное количество дикторов в вашем датасете.
    *   `training.mode`: Установите в `train`.
    *   `training.checkpoint_dir`: Укажите, куда сохранять чекпоинты.
    *   `project.experiment_name`: Задайте имя эксперимента.
    *   Проверьте и настройте другие параметры обучения (эпохи, скорости обучения, размеры батчей, параметры модели и т.д.).

2.  **Запустите обучение:**
    ```bash
    python scripts/run.py --config config/train_base_config.yaml --mode train
    ```
    Процесс обучения будет логироваться в консоль и в файл, указанный в `training.log_file_path`. Чекпоинты будут сохраняться в `training.checkpoint_dir/project.experiment_name/`.

### Дообучение (Fine-tuning) Модели

1.  **Подготовьте данные для fine-tuning**: Аудиозаписи целевого диктора в отдельной папке.
2.  **Настройте конфигурационный файл** (например, `config/finetune_userX_config.yaml`):
    *   `data.fine_tune_user_data_dir`: Укажите путь к аудиоданным целевого диктора.
    *   `model.num_speakers`: Может оставаться таким же, как у базовой модели, или быть адаптировано, если fine-tuning подразумевает добавление нового ID диктора (требует доработки логики эмбеддингов). Для StarGAN-VC обычно используется ID существующего диктора из базовой модели или специальный ID, если модель это поддерживает.
    *   `training.mode`: Установите в `fine_tune`.
    *   `fine_tuning.base_model_checkpoint_path`: Укажите путь к чекпоинту **генератора** предобученной базовой модели.
    *   Настройте параметры в секции `fine_tuning` (количество эпох, скорости обучения - обычно ниже, чем при базовом обучении).
    *   `project.experiment_name`: Задайте новое имя эксперимента для fine-tuning (например, "finetune_userX").

3.  **Запустите дообучение:**
    ```bash
    python scripts/run.py --config config/finetune_userX_config.yaml --mode fine_tune
    ```
    Модель будет дообучаться на предоставленных данных. Чекпоинты будут сохраняться в соответствии с новым `experiment_name`.

### Преобразование Голоса из Файла

1.  **Настройте конфигурационный файл** (например, `config/inference_config.yaml`):
    *   `training.mode`: Установите в `inference_file`.
    *   В секции `inference_file`:
        *   `input_wav_path`: Путь к исходному аудиофайлу.
        *   `output_wav_path`: Куда сохранить результат.
        *   `conversion_direction`: Выберите направление ('male_to_female', 'female_to_male', 'specific').
        *   В зависимости от `conversion_direction`, укажите пути к соответствующим моделям генератора (`fine_tuned_model_path_female_generator`, `fine_tuned_model_path_male_generator`, `specific_generator_checkpoint_path`) и ID целевых дикторов.
    *   Убедитесь, что `model.vocoder.checkpoint_path` указывает на корректный вокодер.

2.  **Запустите преобразование:**
    ```bash
    python scripts/run.py --config config/inference_config.yaml --mode inference_file
    ```
    Результат будет сохранен в `inference_file.output_wav_path`.

### Преобразование Голоса в Реальном Времени

1.  **Настройте конфигурационный файл** (например, `config/realtime_config.yaml`):
    *   `training.mode`: Установите в `inference_realtime`.
    *   В секции `inference_realtime`:
        *   `input_device_index`, `output_device_index`: Установите ID ваших аудиоустройств. Если оставить -1, скрипт предложит выбрать их интерактивно.
        *   Настройте `conversion_direction` и связанные пути к моделям/ID дикторов, аналогично `inference_file`.
        *   `noise_suppression_enabled`: `true` или `false`.
        *   `buffer_size_samples`: Размер буфера (влияет на задержку).
    *   Убедитесь, что `model.vocoder.checkpoint_path` указывает на корректный вокодер.

2.  **Запустите преобразование в реальном времени:**
    ```bash
    python scripts/run.py --config config/realtime_config.yaml --mode inference_realtime
    ```
    При первом запуске, если ID устройств не указаны, скрипт выведет список доступных устройств и попросит ввести их ID. Для остановки используйте Ctrl+C.

## Оценка Качества

В проекте предусмотрена возможность оценки качества сгенерированного аудио с использованием объективных метрик. Файл `src/evaluation.py` содержит функции для вычисления:

*   **MCD (Mel Cepstral Distortion)**: `calculate_mcd(...)`
*   **F0-RMSE (F0 Root Mean Squared Error)**: `calculate_f0_rmse(...)`

**На данный момент эти функции являются плейсхолдерами** и возвращают случайные значения. Для реальной оценки их необходимо имплементировать, используя соответствующие алгоритмы (например, извлечение MFCC, DTW для выравнивания, извлечение F0 с помощью `librosa.pyin`).

Параметры для этих метрик можно задать в секции `evaluation` конфигурационного файла. Во время обучения с валидацией, если `val_loader` предоставлен, эти метрики будут рассчитываться (их плейсхолдеры), и лучшая модель может сохраняться на основе `evaluation.primary_metric`.

## Текущие Ограничения и TODO

*   **Реализация метрик**: Функции MCD и F0-RMSE в `src/evaluation.py` являются плейсхолдерами.
*   **Логика вокодера**: Интеграция HiFi-GAN в `src/models/vocoder.py` является структурным плейсхолдером. Необходимо загрузить реальную модель и реализовать инференс. То же касается `RealTimeVoiceConverter`, который использует этот вокодер.
*   **Потоковая обработка в Real-time**: Логика в `RealTimeVoiceConverter._audio_callback` для потоковой обработки аудио (буферизация, оконное преобразование мел-спектрограмм, инференс модели, вывод через вокодер) является сложной задачей и на данный момент представлена упрощенным плейсхолдером (passthrough/тишина). Требует значительной доработки для достижения низкой задержки и качественного преобразования.
*   **Подготовка данных для `VoiceDataset`**: Функция `load_data_for_dataset` в `scripts/run.py` является плейсхолдером и должна быть заменена реальной логикой парсинга метаданных или сканирования директорий для формирования списков файлов и ID дикторов для обучения и валидации.
*   **Обработка различных Speaker ID**: Текущая реализация эмбеддингов дикторов в `Trainer` и `inference.py` предполагает простое создание `nn.Embedding`. Для более сложных сценариев (например, добавление новых дикторов при fine-tuning без изменения `num_speakers` в базовой модели) может потребоваться более гибкая система управления эмбеддингами.
*   **Тесты**: Отсутствуют юнит-тесты и интеграционные тесты.
*   **Продвинутые архитектуры/техники**: Не реализованы более современные подходы или улучшения StarGAN-VC (например, использование фонем, более сложные блоки внимания и т.д.).

Этот проект представляет собой основу, которую можно расширять и улучшать.
